## What is Yosegi

A desktop application (Go + Wails + Vue 3) that creates photo mosaics: it divides a target image into a grid, finds the best-matching image from a user's collection for each cell (by average color), and stitches them into a final mosaic JPEG.

## Development Commands

**Prerequisites:** Go 1.25+, Node 20+, Wails CLI v2

```bash
# Run in dev mode (hot reload for both Go and Vue)
wails dev

# Build distributable for current platform (output: build/bin/Yosegi.app on macOS)
wails build

# Frontend only (from frontend/)
npm install
npm run dev       # Vite dev server
npm run build     # Produces dist/ (embedded by Wails)
```

There are no lint or test commands configured in this project.

## Architecture

### Backend (Go)

- **`main.go`** — Wails bootstrap only
- **`app.go`** — Wails-bound methods exposed to the frontend: file pickers (`SelectTargetImage`, `SelectCollectionFolder`), image preview generation, `GenerateCollage` orchestration. Emits progress events via `EventsEmit(ctx, "progress", message)`.
- **`imaging.go`** — Core image processing pipeline:
  1. `analyseImage()` — divides target into N×N grid, computes average RGB per cell
  2. `readCollection()` — loads collection folder, computes average color per image (parallel goroutines)
  3. `mapCollectionToTarget()` — for each grid cell, finds closest collection image by Euclidean RGB distance (parallel)
  4. `buildCollage()` — resizes matched images to tile size, stitches mosaic (parallel, with an image cache to avoid re-reading disk)
- **`models/`** — `Pixel` (RGB) and `AveragedImageData` (image path + pixel metadata) structs

### Frontend (Vue 3 + TypeScript)

- **`frontend/src/App.vue`** — owns all app state and screen routing (5-step wizard). Listens for `"progress"` events from Go and drives UI transitions.
- **`frontend/src/components/`** — one component per wizard step plus shared primitives:
  - `LandingScreen.vue`, `StepCollection.vue`, `StepReferenceImage.vue`, `StepSettings.vue`, `StepProcessing.vue`
  - `FileInput.vue`, `ImagePreview.vue`, `TitleBar.vue` (macOS drag region)
- **`frontend/wailsjs/`** — auto-generated Go↔JS bindings (do not edit manually). Regenerated by `wails dev`/`wails build`.

### Go↔Frontend Bridge

Frontend calls Go via `window.go.*` (generated bindings in `wailsjs/go/`). Go emits real-time progress to frontend using Wails event system. All image I/O happens on the Go side; the frontend only passes paths and settings.

### Theming

Global CSS variables in `frontend/src/style.css` define light/dark themes (warm brown palette). Components use scoped CSS referencing those variables. Theme toggle state lives in `App.vue`.

### Performance Logging

`GenerateCollage` writes per-step timing to `yosegi_perf.log` in the working directory.

## Key Conventions

- Image working directories (`images/target/`, `images/collections/`, `images/output/`) are gitignored and created at runtime.
- The `wailsjs/` directory is auto-generated — never edit it manually.
- Frontend `dist/` is produced by `wails build` and embedded into the Go binary; do not commit it.
